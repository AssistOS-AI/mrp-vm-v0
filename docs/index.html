Repository overview

MRP-VM v0 Documentation

This repository now contains the implemented MRP-VM v0 runtime together with the normative DS set, the HTML documentation shell, default KU assets, and verification scripts.

Current repository role

The repository now ships the MRP-VM runtime under src/, the optional server and operational UI adapter under top-level server/, organized tests under tests/, and the architectural specification suite that remains the implementation contract.

Source of truth: the implementation-facing rules live in the DS matrix, with DS001-coding-style.md as the coding-style authority.

Documentation surfaces

Runtime architecture

The implemented MRP-VM execution model, command surfaces, runtime boundaries, hosting layer, four-page operational UI, model-routing strategy, bounded reasoning surfaces, and document-scale planning surface are summarized in Runtime Architecture and specified in DS002 through DS031.

Audit trail

Architectural decisions and unresolved design choices now live directly inside each DS file under the Decisions & Questions section so implementation review stays local to the affected subsystem.

Implementation scope

The documented scope is strictly MRP-VM v0: SOP Lang, state families, native commands, external interpreters including `HumanLikeReasoner`, `AdvancedReasoner`, and `DocumentScalePlanner`, trace, test seams, and implementation constraints.

Implementation posture

The implementation follows the Node.js .mjs posture defined in DS001. The runtime tree under src/ is intentionally modular, the hosting layer and UI templates, styles, and browser logic live under server/, the default KUs live under data/default/ in declaration-style SOP Lang, and the test suite mirrors subsystem boundaries under tests/.

The current server surface is chat-first but operationally transparent: a compact conversation shell with session and authority status in the header is complemented by dedicated Traceability, KB Browser, and tab-structured Settings pages. Session origin, auth mode, effective role, and API-key-backed authority remain part of the documented hosting boundary. DS004 and DS021 remain proposal-only surfaces, while DS022 and DS023 define the implemented server/API/UI and LLM-routing boundaries. DS026 through DS031 define the implemented bounded reasoning and document-scale planning surfaces centered on `HumanLikeReasoner`, `AdvancedReasoner`, `DocumentScalePlanner`, and their routing guidance.
